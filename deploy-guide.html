02 · 安全控制
安全控制建议汇总
12 项安全最佳实践，覆盖从部署到运行的全生命周期。
01
不对外暴露 OpenClaw
除非必须，否则仅使用 localhost。对外暴露时通过防火墙严格控制访问，添加 WAF 防护及 Ratelimit 限制，记录访问日志并监控异常。
02
定期扫描并修复漏洞
扫描 Host 的漏洞并及时安装更新补丁，定期升级 OpenClaw 版本。不使用非官方来源的安装包或升级包。
03
使用隔离的环境运行
将 OpenClaw 部署在独立的虚拟机中，通过 VPC 限制可访问的网络及域名，让 OpenClaw 仅在沙箱功能运行 Tool。
04
谨慎安装 Skills
仅使用可信来源的 Skills，检查 Skills 的安全性和影响。企业应维护自己的 Skills 仓库并审查后再入库，在沙箱中运行并分析执行行为。
05
谨慎处理外部数据
尽可能只访问内部系统及数据，避免访问防火墙外部的系统及数据。对外暴露时记录访问日志并监控异常。
06
仅授予最小权限
使用非 Root 用户运行，通过 tools.allow/deny 仅允许调用所需的 Tool。仅授予必要的外部系统权限，对于敏感操作在 OpenClaw 外部实现人工确认机制。
07
检查不安全的配置
从最小配置集开始，修改后执行 openclaw security audit 命令检查不安全的配置。对配置文件进行完整性监控。
08
安全管理密钥
为 OpenClaw 所使用的凭据添加 IP 限制，降低凭据丢失的影响。使用 Secrets Manager 安全存储密钥，走轮转机制。保护 .openclaw 目录，仅允许 OpenClaw 用户读写。
09
使用身份委托避免越权访问
OpenClaw 访问外部系统时应传播用户身份，使用最终用户的身份访问外部系统。若 OpenClaw 被授予了代身份访问外部系统的权限，应为 OpenClaw 的访问设置独立控制，避免通过此类权限提升。
10
部署运行时安全监控
部署终端安全工具（如 GuardDuty）监控 Host 安全事件，对异常行为及时告警。添加安全事件的自动化响应，例如自动隔离 Host 的网络。
11
维护 Agent 资产并监控未授权安装
扫描主机是否有监听 OpenClaw 常用端口（如 18789），扫描主机是否存在 .openclaw 目录，及时记录合规的 Agent 并清理未经批准的安装。
12
将 AI Agent 纳入信息安全治理体系
制定 AI Agent 相关安全策略，标准和流程。将 AI Agent 常见风险纳入安全意识培训，对你公司允许使用的 AI 网站或工具做防火墙和终端安全工具进行封控。

02 · 安装指南
安装 OpenClaw
OpenClaw 不只是聊天软件——它是一个能调用工具、执行代码、控制设备、主动推送的 AI 智能体。以下是各环境的安装方式。
本地安装（macOS / Linux）
🖥️ 一行命令，本地启动
curl -fsSL https://openclaw.com/install | bash
openclaw start
安装完成后访问 http://localhost:18789 即可使用。支持 Bedrock、OpenAI、Gemini 等多种模型后端。
AWS EC2 部署
☁️ CloudFormation 一键部署
GitHub → aws-samples/sample-OpenClaw-on-AWS-with-Bedrock
点击 Launch Stack → 选择 Region → 约 8 分钟上线
自动创建 VPC、EC2（Graviton ARM）、IAM Role、VPC Endpoints。通过 SSM 端口转发安全访问，无需暴露公网端口。
Lightsail 快速部署
⚡ 最简单的云上方案
在 AWS Lightsail 控制台选择 OpenClaw 镜像，几分钟即可启动。预配置 Bedrock 作为默认 AI 模型提供商，开箱即用。适合个人和小团队快速体验。
Windows 安装
🪟 WSL 或 Docker
Windows 用户可通过 WSL（Windows Subsystem for Linux）安装，或使用 Docker 容器运行。推荐 WSL 2 + Ubuntu 24.04 环境，安装方式与 Linux 相同。
03 · 开发工具
安装 Kiro IDE & CLI
Kiro 是 AI 驱动的开发环境，也是 OpenClaw 的最佳搭档。IDE 用于开发，CLI 用于终端操作。
Kiro IDE
🧠 AI 驱动的 IDE
基于 Code OSS，支持 VS Code 扩展。内置 AI Agent、Spec 驱动开发、Steering 文件、MCP 集成等能力。
下载地址：kiro.dev/downloads
支持 macOS / Windows / Linux
Kiro CLI
⌨️ 终端里的 AI 助手
把 AI 能力带到终端。支持自然语言对话、MCP、Steering、自定义 Agent。
curl -fsSL https://cli.kiro.dev/install | bash
Windows: winget install Amazon.KiroCLI
04 · 协同工作
Kiro + OpenClaw 联动
Kiro 是大脑，OpenClaw 是双手。Kiro 负责思考和编排，OpenClaw 负责执行和交互。两者结合，释放 GenAI 的全部潜力。
🔗
MCP 集成
在 Kiro 中配置 .kiro/settings/mcp.json，连接 OpenClaw 的 MCP Server。Kiro 可以直接调用 OpenClaw 上安装的 Agent 和 Skills。
🛠️
远程运维
Kiro 通过 SSM 端口转发连接云上的 OpenClaw，执行部署、配置修改、故障排查等运维操作。全程加密，无需暴露公网端口。
🤖
Agent 编排
OpenClaw 不只是聊天——它是一个完整的 AI Agent 平台。通过 Skills 扩展能力，通过 MCP 连接外部工具，通过 Kiro 编排复杂工作流。能发邮件、控设备、执行代码、主动推送。
典型工作流
Kiro IDE → SSM 端口转发 → OpenClaw Agent → MCP / Skills → AWS 服务 / 外部工具
开发者在 Kiro 中用自然语言下达指令 → Kiro 通过 SSM 安全连接 OpenClaw → OpenClaw 调用 Skills 和 MCP 工具执行任务 → 结果回传到 Kiro 展示。整个过程对话即操作，安全可审计。
05 · 安全控制
安全控制建议汇总
12 项安全最佳实践，覆盖从部署到运行的全生命周期。
01
不对外暴露 OpenClaw
除非必须，否则仅使用 localhost。对外暴露时通过防火墙严格控制访问，添加 WAF 防护及 Ratelimit 限制，记录访问日志并监控异常。
02
定期扫描并修复漏洞
扫描 Host 的漏洞并及时安装更新补丁，定期升级 OpenClaw 版本。不使用非官方来源的安装包或升级包。
03
使用隔离的环境运行
将 OpenClaw 部署在独立的虚拟机中，通过 VPC 限制可访问的网络及域名，让 OpenClaw 仅在沙箱功能运行 Tool。
04
谨慎安装 Skills
仅使用可信来源的 Skills，检查 Skills 的安全性和影响。企业应维护自己的 Skills 仓库并审查后再入库，在沙箱中运行并分析执行行为。
05
谨慎处理外部数据
尽可能只访问内部系统及数据，避免访问防火墙外部的系统及数据。对外暴露时记录访问日志并监控异常。
06
仅授予最小权限
使用非 Root 用户运行，通过 tools.allow/deny 仅允许调用所需的 Tool。仅授予必要的外部系统权限，对于敏感操作在 OpenClaw 外部实现人工确认机制。
07
检查不安全的配置
从最小配置集开始，修改后执行 openclaw security audit 命令检查不安全的配置。对配置文件进行完整性监控。
08
安全管理密钥
为 OpenClaw 所使用的凭据添加 IP 限制，降低凭据丢失的影响。使用 Secrets Manager 安全存储密钥，走轮转机制。保护 .openclaw 目录，仅允许 OpenClaw 用户读写。
09
使用身份委托避免越权访问
OpenClaw 访问外部系统时应传播用户身份，使用最终用户的身份访问外部系统。若 OpenClaw 被授予了代身份访问外部系统的权限，应为 OpenClaw 的访问设置独立控制，避免通过此类权限提升。
10
部署运行时安全监控
部署终端安全工具（如 GuardDuty）监控 Host 安全事件，对异常行为及时告警。添加安全事件的自动化响应，例如自动隔离 Host 的网络。
11
维护 Agent 资产并监控未授权安装
扫描主机是否有监听 OpenClaw 常用端口（如 18789），扫描主机是否存在 .openclaw 目录，及时记录合规的 Agent 并清理未经批准的安装。
12
将 AI Agent 纳入信息安全治理体系
制定 AI Agent 相关安全策略，标准和流程。将 AI Agent 常见风险纳入安全意识培训，对你公司允许使用的 AI 网站或工具做防火墙和终端安全工具进行封控。

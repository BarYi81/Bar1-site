Kiro + OpenClaw
AWS 运维助理

用自然语言管理 AWS 资源。Kiro 负责思考和编排，OpenClaw 负责执行和交互。 通过 SSM 加密隧道连接，多层权限控制，全链路可审计——安全、可控、高效。

Kiro IDE Kiro CLI OpenClaw AWS MCP SSM Terraform CloudTrail

协作架构

完整的协作链路，每一层都有明确的职责和安全边界：

开发者 → Kiro IDE / CLI → SSM 加密隧道 → OpenClaw Agent → MCP / Skills → AWS 服务
  • 交互层：开发者在 Kiro 中用自然语言描述意图
  • 传输层：SSM 端口转发，IAM 认证，全程加密，零公网暴露
  • 执行层：OpenClaw Agent 调用 MCP 工具执行操作
  • 资源层：EC2、VPC、RDS、S3、Bedrock 等 AWS 服务

安全与可控性

AI 驱动运维最大的顾虑是"会不会失控"。这套方案从架构层面解决了这个问题：

🛡️ Human-in-the-Loop

资源变更类操作的 MCP autoApprove 设为空，每次操作暂停等待人工确认。AI 提议，人决策——比直接敲命令多了一层审查缓冲。

🔒 网络零暴露

OpenClaw 绑定 localhost，安全组入站规则为空。所有访问通过 SSM 加密隧道，不需要 SSH 密钥，不需要 VPN，攻击面为零。

🔑 权限分层

IAM 最小权限 → MCP 操作审批 → OpenClaw sandbox 隔离 → tools.allow/deny 白名单 → SCP 组织级管控。五层纵深防御，逐级拦截。

📋 全链路审计

CloudTrail 记录 API 调用，SSM 记录 session 日志，OpenClaw 记录对话历史，MCP 记录工具调用。出了问题可以完整回溯。

动手实验：配置 Kiro + OpenClaw 协作

前提：你已经通过前面的实验完成了 Kiro IDE 安装和 OpenClaw 在 AWS EC2 上的部署。接下来我们打通两者的协作链路，并完成一个小实验。

Step 1：建立 SSM 端口转发

通过 SSM 在本地和 EC2 之间建立加密隧道，将 OpenClaw 的 18789 端口映射到本地：

终端（CMD / PowerShell）
aws ssm start-session \
  --target <你的EC2实例ID> \
  --region us-east-1 \
  --document-name AWS-StartPortForwardingSession \
  --parameters '{"portNumber":["18789"],"localPortNumber":["18789"]}'

看到 Port 18789 opened for sessionId ... 表示连接成功。保持这个终端窗口不要关闭。

💡 Windows 用户注意：parameters 的 JSON 在 PowerShell 中需要转义。建议将 JSON 保存为文件，用 --parameters file://ssm-params.json 引用。

Step 2：验证连通性

打开浏览器访问 http://localhost:18789，应该能看到 OpenClaw 的 Gateway Dashboard。

或者在 Kiro 的终端中验证：

Kiro 终端
curl http://localhost:18789
# 返回 200 OK 表示连通

Step 3：配置 MCP 工具

在 OpenClaw 服务器上配置 AWS MCP Server，让 OpenClaw 具备 AWS 资源管理能力。

通过 SSM 在 EC2 上编辑 MCP 配置文件：

EC2 服务器上（通过 SSM session）
cat > ~/.kiro/settings/mcp.json << 'EOF'
{
  "mcpServers": {
    "aws-knowledge-mcp-server": {
      "url": "https://knowledge-mcp.global.api.aws",
      "type": "http",
      "disabled": false,
      "autoApprove": ["*"]
    },
    "aws-documentation-mcp-server": {
      "command": "uvx",
      "args": ["awslabs.aws-documentation-mcp-server@latest"],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR"
      },
      "disabled": false,
      "autoApprove": ["*"]
    },
    "aws-ccapi-mcp-server": {
      "command": "uvx",
      "args": ["awslabs.ccapi-mcp-server@latest"],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR",
        "AWS_REGION": "us-east-1"
      },
      "disabled": false,
      "autoApprove": []
    },
    "aws-api-mcp-server": {
      "command": "uvx",
      "args": ["awslabs.aws-api-mcp-server@latest"],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR",
        "AWS_REGION": "us-east-1"
      },
      "disabled": false,
      "autoApprove": []
    },
    "terraform-mcp-server": {
      "command": "uvx",
      "args": ["awslabs.terraform-mcp-server@latest"],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR"
      },
      "disabled": false,
      "autoApprove": ["*"]
    }
  }
}
EOF
🔒 安全提示：aws-ccapi-mcp-server 和 aws-api-mcp-server 的 autoApprove 设为空数组，意味着每次资源变更操作都需要人工确认。这是安全设计的核心。

Step 4：在 Kiro 中连接 OpenClaw

在 Kiro IDE 中，你可以通过终端直接与 OpenClaw 交互。SSM 端口转发建立后，OpenClaw 的所有能力都可以通过 localhost:18789 访问。

在 Kiro 终端中测试 OpenClaw 的 AI 能力：

Kiro 终端
# 通过 SSM 在 EC2 上执行命令
aws ssm send-command \
  --instance-ids <你的EC2实例ID> \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=["openclaw status"]' \
  --region us-east-1

# 查看命令执行结果
aws ssm get-command-invocation \
  --command-id <上一步返回的CommandId> \
  --instance-id <你的EC2实例ID> \
  --region us-east-1

看到 OpenClaw 的状态信息（版本、模型、渠道等）表示 Kiro 和 OpenClaw 已经成功联通。

Step 5：实战演练 — 用 Kiro + OpenClaw 搭建静态网站

现在来做一个完整的实战：通过 Kiro 指挥 OpenClaw，在 AWS 上搭建一个静态网站。

5.1 创建 S3 桶并开启静态网站托管

在 Kiro 终端中，通过 SSM 让 OpenClaw 的 EC2 执行 AWS CLI：

Kiro 终端 → SSM → EC2
# 创建 S3 桶（替换 my-bar1-website 为你的桶名）
aws ssm send-command \
  --instance-ids <EC2实例ID> \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=["aws s3 mb s3://my-bar1-website --region us-east-1"]' \
  --region us-east-1

5.2 创建网站文件

让 OpenClaw 在 EC2 上生成一个简单的 HTML 页面：

Kiro 终端 → SSM → EC2
aws ssm send-command \
  --instance-ids <EC2实例ID> \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=["cat > /tmp/index.html << EOF
<!DOCTYPE html>
<html>
<head><title>My First Kiro+OpenClaw Site</title></head>
<body style=\"background:#0b1120;color:#e8ecf4;font-family:sans-serif;
  display:flex;align-items:center;justify-content:center;height:100vh\">
  <div style=\"text-align:center\">
    <h1 style=\"color:#F0A030\">Hello from Kiro + OpenClaw!</h1>
    <p>This site was deployed by AI — securely, through SSM.</p>
  </div>
</body>
</html>
EOF"]' \
  --region us-east-1

5.3 上传到 S3 并配置静态网站

Kiro 终端 → SSM → EC2
# 上传 HTML 文件
aws ssm send-command \
  --instance-ids <EC2实例ID> \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=[
    "aws s3 cp /tmp/index.html s3://my-bar1-website/index.html --content-type text/html",
    "aws s3 website s3://my-bar1-website --index-document index.html",
    "aws s3api put-bucket-policy --bucket my-bar1-website --policy {\"Version\":\"2012-10-17\",\"Statement\":[{\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::my-bar1-website/*\"}]}",
    "echo WEBSITE_DEPLOYED"
  ]' \
  --region us-east-1

5.4 验证网站

部署完成后，访问你的静态网站：

浏览器
http://my-bar1-website.s3-website-us-east-1.amazonaws.com

看到 Hello from Kiro + OpenClaw! 就表示实验成功。

🎉 恭喜！你刚刚完成了一次完整的 Kiro + OpenClaw 协作：Kiro 编排指令 → SSM 安全传输 → OpenClaw EC2 执行 AWS CLI → S3 静态网站上线。全程零公网暴露，全链路可审计。

Step 6：清理资源（可选）

实验完成后，清理创建的资源避免产生费用：

Kiro 终端 → SSM → EC2
aws ssm send-command \
  --instance-ids <EC2实例ID> \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=[
    "aws s3 rm s3://my-bar1-website --recursive",
    "aws s3 rb s3://my-bar1-website",
    "echo CLEANUP_DONE"
  ]' \
  --region us-east-1

最佳实践

  • IaC 优先：所有基础设施变更通过 Terraform/CloudFormation 代码实现，不直接用 CLI 修改
  • 最小权限：IAM Role 只授予当前任务所需的最小权限，定期审查和收缩
  • 操作审批：资源变更类 MCP 工具保持 autoApprove 为空，每次人工确认
  • 定期审计：定期检查 CloudTrail 日志、安全组规则、IAM 策略
  • 沙箱执行：OpenClaw Agent 在 sandbox 模式下运行，限制文件系统和网络访问
  • 版本控制：所有配置文件和 IaC 代码纳入 Git 管理
